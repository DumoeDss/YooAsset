# 资源部署

在资源补丁包构建成功之后，需要将补丁包传输到CDN服务器上。

如果是本地测试，可以在本地创建一个WEB服务器，然后将补丁包拷贝到WEB服务器下。

**按照游戏版本目录部署**

在业务开发过程中，每个游戏版本实际都会创建一个SVN分支，该分支工程内每次构建的补丁包上传到对应的CDN目录下即可。

````
CDN
└─android
    ├─v1.0
    ├─v1.1
    └─v2.0
└─iphone
    ├─v1.0
    ├─v1.1
    └─v2.0   
````

**游戏版本说明**

v1.0 代表的是游戏版本，不是资源版本。在没有更换安装包的前提下，不需要递增这个游戏版本。每次生成的补丁包只需要覆盖掉当前游戏版本目录下即可。
